You are building a two-stage computer vision pipeline. Stage one runs YOLOv9c, a real-time object detector on every frame from the BDD100K driving dataset. It outputs bounding boxes around every detected object: cars, pedestrians, cyclists, traffic lights. Stage two takes those bounding boxes and feeds them as prompts into SAM (Segment Anything Model), which draws pixel-precise masks around each detected object. The entire two-stage process completes in under 28ms per frame on an A100 GPU.

You then export the trained YOLOv9 model to ONNX format, a hardware-agnostic model format and serve it behind a FastAPI REST server, containerized in Docker, orchestrated by Kubernetes, and monitored via Grafana. The 45% cost reduction comes specifically from ONNX Runtime replacing PyTorch's full runtime at inference time, which means you can serve on a cheaper EC2 instance with no accuracy loss.
